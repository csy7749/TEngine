# UI 性能基准与优化指南

## 基准建议

- 记录窗口打开耗时、首次渲染时间、内存峰值。
- 采样 UI 绑定更新次数与绑定分配频率。
- 对常驻窗口与弹窗分别采集数据。

## 优化方向

- 避免频繁创建/销毁窗口：优先使用 `Cache` 模式。
- 降低绑定更新频率：减少高频属性变更，合并通知。
- 统一颜色/本地化配置：减少运行时查找与反射。
- 对列表/网格类控件做对象池管理。

## 常见瓶颈

- 复杂层级的 `Transform.Find` 开销大，建议在 `OnCreate` 完成缓存。
- 异步加载过多资源导致卡顿，建议预加载关键 UI。

## 监控建议

- 定期输出 UIManager 活动窗口数量。
- 对 RedDot 更新进行次数统计，避免级联更新过多。
